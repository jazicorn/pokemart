# Log
> feature tracker

## About

I created a table to keep track of what I work on in the project.
I quickly realized that it would take some time to completely set up the project for both development and production, plus implement all the features I wanted.

### Options
> Example of tracker table inputs

- **ID**
  - Ex. 001

- **Status** | [emoji's](https://gist.github.com/rxaviers/7360908)
  - :-1: &nbsp; **Incomplete** 
    - *(Not Started)*
  - :wave: &nbsp; **Staged** 
    - *(Setup Complete:)* 
    - *(Ex. Packages;Imports;Documentation;etc.)*
  - :pray: &nbsp; **Work-In-Progress** 
    - *(Started Coding)* 
  - :tada: &nbsp; **Complete** 
    - *(Documented & Finished)*

- **Category**
  - Project Management (PM)
  - Frontend (Client)
  - Backend (Server)
  - Devops (DevOps)

- **Feature**
  - React
  - CSS
  - Nodejs
  - Eslint
  - etc.
